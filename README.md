# Curso de Node.js 

# [Aula 01: Baixar e instalar o Node.js](https://jornadadodev.com.br/cursos/back-end/nodejs/baixar-e-instalar-o-node-js-testar-e-instalar-ferramentas-extras)

Essa aula teve como princípio ensinar ao aluno instalar as ferramentas necessárias para o desenvolvimento com node.js, orientando cada processo minuciosamente e com clareza, facilitando o processo de instalação. Nessa aula, pude conhecer a ferramenta e como ela funcionava, e devido à dinamicidade do tutorial, o processo de instalação foi rápido.


# [Aula 02: Criar projeto API REST com Express, rotas e Nodemon](https://jornadadodev.com.br/cursos/back-end/nodejs/criar-projeto-api-rest-express-rotas-nodemon)

Essa aula foi essencial para a próxima etapa do curso, em que realmente é iniciado, mesmo que aos poucos, práticas de desenvolvimento com node.js. A partir dela, o aluno conhece o Express — framework web para Node.js que facilita e acelera a criação de aplicativos e APIs back-end, oferecendo recursos como roteamento, tratamento de requisições HTTP e middleware —. Adiante, é orientado ao aluno a importar o Express no Visual Studio Code — editor de código-fonte gratuito e de código aberto desenvolvido pela Microsoft — através de comandos próprios do Javascript.
Ainda nessa aula, pude importar o Express, criar minha primeira API, e definir a porta do servidor, assim como foi orientado nessa aula do curso de Node.js.


# [Aula 03: Criar rotas GET com Express, refatorar código e extensão JSON Viewer](https://jornadadodev.com.br/cursos/back-end/nodejs/criar-rotas-get-com-express-refatorar-codigo-extensao-json-viewer)

Nessa aula o ensino voltado a criação de rotas com Express foi crucial para a definição de como a aplicação responderá a requisições HTTP vindas do usuário. Nela, pude aprender como criar um servidor utilizando o "server.js", juntamente com a insstalação do Express. Ainda nesse contexto, pude aprender comandos como o res.send — comando responsável por disponibilizar uma resposta ao usuário ao acessar tal endereço —, e aplicar ele na base da API. E, além disso, pude aprimorar a organização dos meus códigos na pasta criada, visando a estruturação da minha API.
